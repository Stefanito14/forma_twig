💡 Pourquoi apprendre Twig ?
Twig est le langage qui transforme vos données en contenu visible sur le site. Le maîtriser vous permet de mieux comprendre ce qui est possible d'automatiser et d'optimiser pour le SEO.

🔧Qu'est-ce que Twig ?

Définition simple

Twig est le langage de template utilisé par Drupal. Il connecte le contenu (données) à l'affichage (HTML) de votre site.

📊 Avant Twig (PHP)

  • Code complexe et difficile à lire
  • Risques de sécurité
  • Collaboration difficile
  • Maintenance compliquée

✅ Avec Twig

  • Syntaxe claire et lisible
  • Sécurisé par défaut
  • Modulaire et réutilisable
  • Collaboration facilitée

🎯 Impact SEO direct

  • Contenu dynamique : Affichage automatique de listes, classements, comparatifs
  • HTML propre : Structure cohérente appréciée par les moteurs de recherche
  • Balises optimisées : H1, H2, métadonnées gérées automatiquement
  • Mise à jour automatique : Contenu toujours à jour via les APIs

💡Exemple Concret d'Usage

🏦 Cas pratique : Afficher les meilleures offres bancaires

Problème : Comment afficher automatiquement un top 3 des meilleures banques selon notre API ?

❌ Méthode manuelle (sans Twig)

  • Copier-coller manuellement les données de l'API
  • Mettre à jour le contenu à chaque changement
  • Risque d'erreurs et d'oublis
  • Temps considérable à chaque modification

✅ Solution Twig (automatisée)

{% set offres = api_data.banks %} {% for offre in offres|slice(0, 3) %} <h3>{{ loop.index }}. {{ offre.name }}</h3> <p>Note : {{ offre.selectra_score }}</p> <p>Prix : {{ offre.price }}€</p> {% endfor %}

Résultat : Top 3 automatique, toujours à jour, avec structure HTML optimisée !

🚀 Avantages pour le SEO

  • Contenu toujours frais et actualisé
  • Structure HTML cohérente (H3, listes, etc.)
  • Aucune intervention manuelle nécessaire
  • Possibilité de créer des variantes selon les critères

📦Variables et Données

🎯 Qu'est-ce qu'une variable ?

Une variable est une donnée préparée par le développeur que vous pouvez afficher dans Twig.

⚠️ Point crucial : Twig ne crée pas les données, il les affiche seulement si elles sont disponibles.

🏦 Variables bancaires

{{ offre.name }} → Nom de l'offre {{ bank.slug }} → Identifiant banque {{ offre.price }} → Prix de la carte {{ offre.selectra_score }} → Note Selectra

🛡️ Variables assurances

{{ assur.insurer.name }} → Nom assureur {{ offre.profiles|first }} → Premier profil {{ garanties.iard }} → Garanties IARD {{ offre.first_price }} → Prix d'entrée

🤝 Collaboration avec les développeurs

Votre rôle : Définir quelles données vous avez besoin d'afficher

Rôle du développeur : Préparer ces données dans le code (preprocess)

Résultat : Variables disponibles dans votre template Twig

🔀Les Conditions en Twig

🎯 Afficher quelque chose seulement si...

Les conditions permettent d'adapter l'affichage selon le contexte.

Structure de base

{% if condition %} <p>Affiché si vrai</p> {% else %} <p>Affiché si faux</p> {% endif %}

💰 Exemple : Prix gratuit

{% if offre.price == 0 %} <p>⭐ {{ offre.name }} : GRATUITE !</p> {% endif %}

Résultat : Affiche le message seulement si le prix est 0

🏆 Exemple : Note Selectra

{% if offre.selectra_score == "A" %} <span class="badge-success">Recommandé</span> {% elseif offre.selectra_score == "B" %} <span class="badge-good">Bon choix</span> {% else %} <span class="badge-neutral">Standard</span> {% endif %}

Résultat : Badge différent selon la note

Comparateur Signification Exemple
== Égal à {% if score == "A" %}
!= Différent de {% if type != "visa" %}
> Supérieur à {% if price > 10 %}
is empty Vide {% if title is empty %}

🔄Les Boucles en Twig

🎯 Répéter une action pour chaque élément

Les boucles permettent d'afficher automatiquement une liste d'éléments.

Structure de base

{% for element in liste %} <p>{{ element.propriete }}</p> {% endfor %}

📊 Exemple : Afficher toutes les offres

{% for offre in bank_offers %} <div class="offre-card"> <h3>{{ offre.name }}</h3> <p>Prix : {{ offre.price }}€</p> <p>Score : {{ offre.selectra_score }}</p> </div> {% endfor %}

Résultat : Une carte pour chaque offre de la liste

🏆 Classement automatique

{% for offre in offres %} <p>#{{ loop.index }} - {{ offre.name }}</p> {% endfor %}

loop.index donne la position (#1, #2, #3...)

🎯 Condition dans une boucle

{% for offre in offres %} {% if loop.index == 1 %} <h2>🥇 {{ offre.name }}</h2> {% else %} <h3>{{ offre.name }}</h3> {% endif %} {% endfor %}

Traitement spécial pour le premier élément

🔧Les Filtres Twig

🎯 Transformer les données avant affichage

Les filtres modifient une donnée avec le symbole |

Syntaxe : {{ variable|filtre }}

Filtre Effet Exemple Résultat
|lower Minuscules {{ "BANQUE"|lower }} banque
|upper Majuscules {{ "banque"|upper }} BANQUE
|title Première lettre maj. {{ "ma banque"|title }} Ma Banque
|length Compter éléments {{ offres|length }} 5 (si 5 offres)
|default() Valeur par défaut {{ titre|default('Pas de titre') }} Si titre vide → "Pas de titre"

🔗 Joindre une liste

{% set tags = ["Score A", "Gratuit", "VISA"] %} {{ tags|join(', ') }}

Résultat : Score A, Gratuit, VISA

🔄 Remplacer du texte

{{ "direct-assurance"|replace({'-': ' '}) }}

Résultat : direct assurance

📊Trier et Manipuler les Listes

🎯 Organiser vos données

Twig permet de trier et filtrer les listes pour un affichage optimal.

🔢 Trier par prix croissant

{% set offres_triees = offres|sort((a, b) => a.price <=> b.price) %} {% for offre in offres_triees %} <p>{{ offre.name }} : {{ offre.price }}€</p> {% endfor %}

Résultat : Offres classées du moins cher au plus cher

🔍 Filtrer selon une condition

{% set offres_gratuites = offres|filter(o => o.price == 0) %} {% for offre in offres_gratuites %} <p>{{ offre.name }} - GRATUIT</p> {% endfor %}

Ne garde que les offres gratuites

🧩 Supprimer les doublons

{% set marques_vues = [] %} {% for produit in produits %} {% if produit.marque not in marques_vues %} <p>{{ produit.marque }}</p> {% set marques_vues = marques_vues|merge([produit.marque]) %} {% endif %} {% endfor %}

Affiche chaque marque une seule fois

⚠️ Limitations Drupal : Certains filtres comme |unique ou |int ne sont pas disponibles dans notre version de Twig.

🔌Intégration de Données API

🎯 Récupérer des données dynamiques

Twig peut afficher des données provenant de nos APIs (banques, assurances, rénovation).

📡 Appel API simple

{% set url = "https://insurance-api.app.production.aws.selectra.io/api/data/banks" %} {% set result = api_complete_request('GET', url, options).data %}

Récupère les banques au sein de notre BDD

🏦 Exemple : Meilleures banques

{% set url = "https://insurance-api.app.production.aws.selectra.io/api/data/banks?sort=best_welcome_promo&filters[show_in_content]=1&direction=desc&per_page=3" %} {% set result = api_complete_request('GET', url, options).data %} {% for bank in result %} <div class="bank-card"> <h3>{{ bank.name }}</h3> <p>Score : {{ bank.best_offer.selectra_score }}</p> </div> {% endfor %}

🛡️ Exemple : Assurances auto

{% set url = "https://insurance-api.app.production.aws.selectra.io/api/data/insurance-offers/car" %} {% set result = api_complete_request('GET', url, options) %} {% for assur in result|slice(0, 5) %} <p>{{ assur.insurer.name }} - {{ assur.selectra_score }}€</p> {% endfor %}

🚀 Avantages pour le SEO

  • Contenu frais : Données toujours à jour automatiquement
  • Pas de maintenance : Aucune intervention manuelle
  • Cohérence : Même structure sur toutes les pages
  • Performance : Filtrage intelligent des données

📝Créer des Variables avec SET

🎯 Stocker et manipuler des données

La fonction set permet de créer des variables temporaires pour vos calculs.

Structure de base

{% set variable_name = valeur %} {{ variable_name }}

🧮 Calculs simples

{% set prix_total = offre.frais + offre.abonnement %} {% set remise = prix_total * 0.1 %} <p>Prix : {{ prix_total }}€</p> <p>Remise : {{ remise }}€</p>

📊 Listes et tableaux

{% set top_banques = ["Boursobank", "Revolut", "N26"] %} {% set scores = {"A": "Excellent", "B": "Bon", "C": "Moyen"} %} {{ top_banques|join(', ') }} {{ scores["A"] }}

🔗 Fusionner des listes

{% set mots_cles_manuels = ['banque', 'carte'] %} {% set mots_cles_api = ['gratuit', 'visa'] %} {% set tous_mots_cles = mots_cles_manuels|merge(mots_cles_api) %} <p>Mots-clés : {{ tous_mots_cles|join(', ') }}</p>

Résultat : banque, carte, gratuit, visa

⚠️ Attention : Vous ne pouvez utiliser que les variables que vous avez définies ou qui ont été préparées par le développeur.

✅Bonnes Pratiques SEO avec Twig

✅ À FAIRE

  • Utiliser des conditions pour adapter l'affichage
  • Créer des classements automatiques avec les boucles
  • Appliquer des filtres pour organiser les données
  • Utiliser default() pour éviter les contenus vides
  • Combiner API + Twig pour du contenu frais

❌ À ÉVITER

  • Utiliser des variables non définies
  • Oublier les conditions de sécurité (is empty)
  • Créer des boucles infinies
  • Ignorer la structure HTML (h1, h2, etc.)
  • Négliger les performances (trop de données)

🎯 Checklist SEO avec Twig

  1. Structure : Utiliser les bonnes balises HTML (h1, h2, p, ul)
  2. Contenu : Afficher des données pertinentes et à jour
  3. Performance : Limiter les données avec des filtres
  4. Robustesse : Prévoir les cas où les données sont vides
  5. Lisibilité : Commenter le code pour la maintenance
